# Project Agent Instructions

## Folder Routing

When working in `frontend/`, read:

- `skills/frontend-design/SKILL.md`
- `context/ui-context.md`
- `context/project-overview.md`
- `context/code-standards.md`

The frontend should be treated as the product UI/design spectrum: React, TypeScript, layout, visual design, accessibility, frontend state, and API integration.

When working in `backend/`, assume FastAPI unless the repository later establishes another backend stack.

When working in `pipeline/`, assume Python data science or data processing unless the repository later establishes another pipeline stack.

## General Rules

- Read the relevant context before editing.
- Prefer existing project patterns over introducing new ones.
- Verify changes before claiming they are done.
- Keep commits focused by folder or feature area.
